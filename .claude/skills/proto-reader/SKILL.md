# Proto Reader

读取产品原型页面（HTML），自动分析页面结构、功能模块、跳转关系，输出结构化需求信息。

## 工作流程

### Phase 1: 页面分析

访问目标 URL，提取以下信息：

1. **页面基本信息**
   - 页面标题（`<title>`）
   - 主要功能区域（header、sidebar、main content、footer）
   - 页面类型（列表页 / 详情页 / 表单页）

2. **功能元素识别**
   - 按钮：新增、编辑、删除、筛选、搜索
   - 表格：列定义、数据行、操作列
   - 表单：输入框、下拉框、日期选择器
   - 链接：跳转、详情、新开页面

3. **跳转关系梳理**
   - 当前页面 → 哪些子页面（列表→详情、列表→新增）
   - 页面间参数传递（ID、状态）
   - 面包屑导航

### Phase 2: 递进遍历（重要！）

**必须递归访问所有子页面**，不能只分析入口页面。

对于发现的子页面（新增、编辑、详情等），递归访问并记录：

1. 从 Vue 源码提取 `router.push` 跳转关系
2. 根据路由名查找对应的 Vue 组件
3. 读取子组件源码，分析页面结构
4. 继续提取跳转关系，直到没有新页面
5. 限制深度：最多 3 层

**跳转关系提取示例**：
```javascript
router.push({ name: 'bean-usage-detail', query: { month, employeeId } })
// → 提取路由名 + 参数
```

### Phase 3: 输出结构化需求

输出格式：

```markdown

- **模块名称**：[从页面标题或导航提取]
- **核心功能**：[一句话描述]

- **URL**: /path
- **类型**: 列表页 / 详情页 / 表单页
```

## 解析规则

### 页面类型识别

| 特征 | 类型 |
|-----|------|
| 包含 `<table>` 或数据列表 | 列表页 |
| URL 包含 `/detail/` 或 `id=` | 详情页 |
| 包含 `<form>` 或大量输入框 | 表单页（新增/编辑） |

### 跳转关系识别

1. 表格操作列：编辑、删除、详情按钮
2. 页面顶部按钮：新增、导出
3. 链接：`href` 包含路径或参数
4. 表单提交：`<form>` action 或 fetch/axios 调用

### 递归策略

- 从入口页面开始
- 发现新页面 URL 加入队列
- 优先访问：列表页 → 详情页、新增页、编辑页
- 限制深度：最多 3 层（列表 → 详情 → 子详情）
- 记录已访问避免重复

## 注意事项

1. **URL 处理**：相对路径转为绝对路径
2. **中文编码**：URL 中的中文需要 encodeURIComponent
3. **SPA 处理**：如果页面使用 Vue/React，需模拟路由变化触发动态内容加载
4. **分页/加载更多**：记录但不深入解析
5. **权限限制**：部分页面可能需要登录，记录即可跳过

## 调用示例

```
用户：帮我分析 http://localhost:5178/store-hanging-application

助手：
1. 访问页面，获取页面结构
2. 发现子页面：详情页、新增页、编辑页
3. 递归访问子页面
4. 输出结构化需求信息
5. （可选）基于此分析，可以调用 prd-generator 生成 HTML 文档
```

## 两种工作模式

### 模式一：单独分析（Standalone）
```
用户：帮我分析 [URL]
       ↓
proto-reader：输出结构化需求信息
       ↓
结束，不自动生成文档
```

### 模式二：联合生成（Chain）
```
用户：帮我分析并生成 [URL] 的 PRD 文档
       ↓
proto-reader：输出结构化需求信息
       ↓
（用户确认后）
       ↓
proto-reader 继续调用 prd-generator 生成 HTML 文档
       ↓
输出文档路径
```

### 如何选择模式

| 场景 | 推荐模式 |
|------|---------|
| 只想了解页面结构 | 单独分析 |
| 需要生成 PRD 文档 | 联合生成 |
| 先分析，再决定是否生成 | 单独分析 → 确认 → 手动调用 prd-generator |
