# Proto to PRD

一体化完成：读取原型页面 → **递进遍历所有子页面** → 分析结构 → 生成面向业务方的交互式 HTML PRD 文档。

## 核心能力

**递进遍历**：从入口页面开始，递归访问所有发现的子页面（详情、新增、编辑等），完整分析整个功能模块的页面结构和跳转关系。

## 两种身份视角

| 身份 | 关注点 | 需要的内容 |
|------|--------|-----------|
| **产品经理** | 业务逻辑、用户交互、业务规则 | ✅ 业务描述 ✅ 筛选条件 ✅ 表格列 ✅ 操作按钮详细说明 ✅ 业务流程图 ✅ 跳转关系 |
| **研发** | 技术实现、组件结构、数据结构 | ⚠️ 参考信息（组件名、数据结构、接口） |

**重要**：生成的文档以**产品经理视角**为主，技术细节作为参考信息标注，不喧宾夺主。

## 工作流程

### Phase 1: 递进遍历

**必须使用 page-analyzer agent 执行递归遍历**：

1. 使用 Agent tool 启动 page-analyzer agent
2. 输入入口页面 URL
3. agent 输出完整的页面结构分析报告（包含所有子页面）

**重要**：page-analyzer agent 会自动：
1. 读取入口 Vue 组件源码
2. 提取所有 router.push 跳转
3. 递归读取每个子页面组件（最多3层）
4. 去重 + 避免循环
5. 输出完整的页面结构和跳转关系

**如果 agent 输出不完整（如缺少子页面），必须重新调用 agent 并检查分析结果**

### Phase 2: 汇总页面结构

1. 汇总所有页面的功能元素
2. 梳理完整的跳转关系图
3. 识别业务场景
4. **判断需要哪些辅助内容**：
   - 业务流程图？（如果跳转逻辑复杂）
   - 模拟器？（如果有计算逻辑）
   - 决策树？（如果有条件分支）

### Phase 3: 生成 PRD 文档

1. 创建模块目录
2. 生成总览页（index.html）和场景页（scene-*.html）
3. **为每个 HTML 页面生成对应的 .md 文件**，用于飞书导入
4. 为每个 HTML 页面的 header 添加 MD 预览按钮
5. 更新首页入口和菜单

## 文档结构规范（面向业务方）

### 总览页（index.html）

```markdown
## 模块信息
- **模块名称**：[中文名称]
- **核心功能**：[一句话描述]
- **适用角色**：[谁使用这个模块]

## 业务流程
[如果是复杂的业务流程，生成流程图]
[如果是简单的增删改查，可以省略]

## 页面结构
[列表所有页面，按层级分组]

## 跳转关系图
[ASCII 或 mermaid 流程图]

## 场景列表
[每个业务场景一个卡片]
```

### 场景页（scene-*.html）

每个场景页必须包含：

```markdown
## 版本信息
| 项目 | 内容 |
|-----|------|
| 当前版本号 | v1.0 |
| 创建日期 | YYYY-MM-DD |
| 负责人 | - |

---

## 业务描述
[这个场景做什么业务，解决什么问题]

---

## 业务流程
[如果涉及多步骤交互，生成流程图]

---

## 筛选条件
| 字段 | 说明 | 控件类型 | 业务说明 |
|------|------|---------|---------|
| 月份 | 选择查询的月份 | 月份选择器 | 支持按指定月份筛选数据 |

---

## 表格列
| 列名 | 说明 | 业务说明 |
|------|------|---------|
| 姓名 | 员工姓名 | 显示员工真实姓名 |

---

## 操作按钮
| 按钮 | 功能 | 详细说明 |
|------|------|---------|
| 查询 | 根据条件搜索 | 支持按城市筛选业务周每日明细结果 |
| 导出 | 导出数据 | 导出搜索条件下所有符合条件的数据，导出字段包括：姓名、性别、岗位... |
| 重置 | 清空条件 | 将指定业务周的配置明细，重置为基础配置的限制 |
```

**重要**：操作按钮的"详细说明"必须包含：
- 按钮触发的具体行为
- 涉及的业务逻辑
- 如果是复杂操作，需要分步骤说明

**重要**：筛选条件和表格列的"业务说明"必须包含：
- 字段的业务含义
- 取值范围或格式要求
- 在业务场景中的作用

## 功能描述规范

### 操作按钮描述模板

```markdown
| 按钮名称 | 触发表单 | 详细行为 | 结果 |
|---------|---------|---------|------|
| 按钮A | 重置表单 | 1. 清空所有输入框<br>2. 恢复默认值<br>3. 提示"已重置" | 回到初始状态 |
```

**示例1 - 导出**：
```
❌ 错误：导出 | 导出数据
✅ 正确：导出 | 导出搜索条件下的所有符合条件的数据 | 导出字段包括：姓名、工号、部门、岗位、本月结算数量、本月提现数量、本月累计消费、实时可用余额，导出格式为 Excel
```

**示例2 - 批量重置**：
```
❌ 错误：重置 | 重置条件
✅ 正确：批量重置 | 将指定业务周的配置明细重置为基础配置的限制 | 1. 确认操作后，目标城市（如成都市）的配置值将恢复为基础配置<br>2. 再次编辑时需显示上次编辑的说明
```

**示例3 - 搜索**：
```
❌ 错误：搜索 | 根据条件搜索
✅ 正确：搜索 | 支持按城市筛选业务周每日明细结果 | 输入城市名称后，系统展示该城市对应业务周的配置数据
```

### 字段描述模板

```markdown
| 字段 | 控件类型 | 取值范围 | 业务说明 |
|------|---------|---------|---------|
| 类型 | 下拉选择 | 1~1200，正整数 | 输入值范围1~1200，必须为正整数 |
| 日期类型 | 下拉选择 | 工作日/周末/节假日 | 工作日指普通工作日，周末指周六周日，节假日需显示具体节日名称如元旦、春节 |
```

## 输出要求

1. **生成位置**：`public/docs-h5/` 目录下（即 `/Users/mac/[项目名]/public/docs-h5/`）
2. **文件命名**：模块文件夹使用中文命名，如 `docs/依赖计算策略/`
3. **场景文件命名**：使用业务场景的英文/拼音标识
4. **HTML header 必须包含 MD 预览按钮**
5. **每个页面生成对应的 .md 文件**

## 注意事项

1. **必须递归访问子页面**：不能只分析入口页面
2. **业务流程图是核心**：如果场景涉及多步骤操作，必须生成流程图
3. **参考信息折叠**：技术实现细节放在可展开的参考区域内
4. **场景命名**：使用业务含义的英文/拼音标识
5. **Vue 组件查找**：优先在 `src/views/` 目录查找
6. **操作按钮必须有详细说明**：不能只写"导出数据"，必须说明导出的具体行为和导出字段
7. **字段必须有业务说明**：不能只写字段名，必须说明业务含义和取值范围
8. **每个发现的子页面必须生成对应场景文档**：如果 BeanAccountMonthly 有3个跳转目标（结算明细/提现明细/消费明细），则必须生成3个场景页，不能遗漏

## 文档完整性检查清单

生成文档后，必须逐项检查：

- [ ] 入口页面文档（index.html + index.md）
- [ ] 每个跳转目标页面对应的场景页（scene-*.html + scene-*.md）
- [ ] 每个场景页包含：筛选条件（全部字段）、表格列（全部列）、操作按钮（详细说明）
- [ ] 每个场景页包含：版本信息、业务描述、业务流程图
- [ ] 菜单 app.js 已更新

**如果发现文档不完整，必须重新执行 page-analyzer agent 进行递归分析**
