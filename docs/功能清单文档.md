# 业务流程与权限管理系统功能清单

## 一、设计理念

### 核心设计思路
- **业务为核心**：先定义业务，再定义业务下的流程
- **流程环节权限配置**：明确环节要使用的RBAC+ABAC+列字段权限
- **快速权限配置**：新建用户→新增角色→指定角色参与业务流程环节
- **流程环节预设权限**：流程和环节本身已配置好所需权限
- **复合权限模型**：角色需同时获得RBAC功能操作权限、ABAC数据访问规则和列字段权限
- **完整职责形成**：所有业务流程环节加在一起，形成岗位的完整工作职责
- **高效配置方式**：可视化配置界面，支持权限模板

### 创新点
- 以业务为核心的流程设计
- 流程环节的预设权限配置
- 支持快速权限配置的角色-业务-流程关联
- 复合权限模型（RBAC+ABAC+列字段权限）
- 完整的岗位职责形成机制
- 权限有效期管理
- 跨组织业务权限支持
- 完整的权限验证流程

## 三、业务流程示例

### 单车调配流程

1. **流程定义**：
   - 流程名称：单车调配流程
   - 流程环节：待调配 → 调配中 → 已完成

2. **环节配置**：
   - 待调配：
     - 所需角色：运营专员、运营总监
     - 权限配置：查看单车列表、调配按钮
     - 数据规则：只显示"待调配"状态的单车
   - 调配中：
     - 所需角色：运营专员、运营总监
     - 权限配置：查看单车列表、完成调配按钮
     - 数据规则：只显示"调配中"状态的单车
   - 已完成：
     - 所需角色：运营总监
     - 权限配置：查看单车列表、审核按钮
     - 数据规则：只显示"已完成"状态的单车

3. **权限配置**：
   - 基于流程环节配置角色权限，而非流程自动分配
   - 运营专员在"待调配"环节可查看待调配状态单车并执行调配操作
   - 运营专员在"调配中"环节可查看调配中状态单车并执行完成操作
   - 运营总监在"已完成"环节可查看已完成状态单车并执行审核操作
   - 权限基于角色在流程环节的配置，流程实例运行时根据当前环节验证权限
