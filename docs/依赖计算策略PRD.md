### 2.3 术语定义

| 术语 | 定义 |
|-----|------|
| 依赖计算策略 | 定义薪酬项之间的计算依赖关系和执行顺序的规则集合 |
| 发放策略 | 定义员工薪酬的发放规则，一个发放策略可对应多个依赖计算策略 |
| 计算规则（规则） | 依赖计算策略下的单个计算单元，包含依赖项、计算公式和结果嵌入 |
| 跨规则依赖 | 当前规则引用前序规则的计算结果 |
| 外部服务 | 系统外部提供的计算服务（如个税申报、负工资抵扣） |

| 数据对象 | 业务实体（员工、订单、门店、客户）及其属性，用于条件判断 |

## 3. 需求背景

### 3.1 业务痛点

在薪酬计算场景中，存在大量复杂的依赖关系：

1. **多工资条联动**：一个员工的薪酬可能涉及多个工资条（如基本工资、绩效、奖金等），某项薪酬的计算可能依赖其他工资条的数据
2. **跨周期计算**：如兼职员工按周预扣个税，但需在月末汇总退补
3. **外部服务依赖**：如个税计算需调用外部税务服务，负工资抵扣需查询员工历史欠款
4. **条件分支逻辑**：同一种薪酬项，根据不同条件（如员工类型、在职状态）可能有不同的计算方式

### 3.2 设计目标

- **可视化配置**：业务方可自主配置复杂的依赖计算规则，无需开发介入
- **执行顺序可控**：明确声明规则间的依赖关系，保证计算顺序正确
- **外部服务集成**：支持调用外部服务（税务、负工资等）并使用其返回结果
- **结果灵活嵌入**：计算结果可嵌入到任意工资条的指定分组中

## 5. 业务场景

### 5.1 场景一：全职员工保底获豆

#### 5.1.1 业务描述

为保障全职员工收入，当员工月度收入（计件提成 + 加班费）低于当地最低工资标准时，由公司补足差额，以「常乐豆」形式发放。

#### 5.1.2 计算逻辑

```
月度收入 = 计件提成 + 超时加班费 + 节假日加班费

if (月度收入 < 最低工资标准):
    保底获豆 = 最低工资标准 - 月度收入
else:
    保底获豆 = 0
```

#### 5.1.3 配置要点

- **依赖项**：计件提成（计件工资条）、超时加班费（加班工资条）、节假日加班费（加班工资条）、最低工资标准（配置参数）
- **计算频率**：月（自然月），当月 M1日结算
- **结果嵌入**：保底获豆 → 常乐豆工资条 → 应发收入分组

### 5.2 场景二：兼职员工个税计算

#### 5.2.1 业务描述

兼职员工按业务周（周一开始）计算薪酬，每周预扣个税，月末汇总进行退补。

#### 5.2.2 计算逻辑

```
第1周预扣 = (基础提成 + 绩效补贴 + 夜班补贴) × 3%
第2周预扣 = (回头客激励 + 服务时长补贴) × 3%
第3周预扣 = (周时长激励 + 节假日补贴) × 3%

月度税前收入 = 基础提成 + 回头客激励 + 周时长激励
实际税额 = 外部服务(TAX，月度税前收入)
预扣税合计 = 第1周预扣 + 第2周预扣 + 第3周预扣

if (预扣税合计 > 实际税额):
    个税退税 = 预扣税合计 - 实际税额
else:
    个税补缴 = 实际税额 - 预扣税合计
```

#### 5.2.3 配置要点

- **规则拆分**：拆分为 4 个规则（前 3 周各自预扣，第 4 周汇总退补）
- **跨规则依赖**：第 4 条规则需显式声明依赖前 3 条规则
- **外部服务**：调用 TAX 申报服务获取实际税额
- **结果嵌入**：
  - 第 1-3 周预扣 → 对应周工资条 → 代扣税分组
  - 第 4 周退税 → 第 4 周工资条 → 应发收入分组
  - 第 4 周补缴 → 第 4 周工资条 → 代扣税分组

### 5.3 场景三：负工资抵扣

#### 5.3.1 业务描述

当员工出现负工资情况时（多发或少扣），需从后续工资中逐步抵扣。根据员工在职状态不同，抵扣方式有所区别：

- **在职员工**：从现金工资条抵扣
- **离职员工**：从常乐豆工资条抵扣（以常乐豆形式体现）

#### 5.3.2 抵扣规则

```
对于每个工资条:
    if (员工.在职状态 == '在职'):
        抵扣来源 = 该工资条现金收入
    else:
        抵扣来源 = 常乐豆工资条收入

    负工资抵扣 = 外部服务(负工资服务, 抵扣来源)
```

#### 5.3.3 配置要点

- **依赖项**：各工资条收入项 + 员工（数据对象，用于判断在职状态）
- **跨规则依赖**：第 2、3 条规则需依赖前序规则计算结果
- **数据对象条件**：根据 员工.在职状态 判断走哪个分支
- **外部服务**：调用负工资抵扣服务，传入收入金额
- **结果嵌入**：
  - 在职员工 → 现金工资条 → 代扣税分组
  - 离职员工 → 常乐豆工资条 → 代扣税分组

### 6.2 核心概念

#### 6.2.1 策略结构

```
依赖计算策略
├── 基本信息
│   ├── 策略名称
│   ├── 关联发放策略
│   └── 有效期
└── 计算规则[]
    ├── 规则1
    │   ├── 规则名称
    │   ├── 计算频率（周期 + 节点）
    │   ├── 依赖项[]
    │   ├── 跨规则依赖[]    ← 显式声明依赖的前序规则
    │   ├── 数据要求（取整模式、数据格式）
    │   ├── 步骤[]
    │   │   ├── 步骤1（结果名称 + 条件分支 + 公式）
    │   │   └── 步骤2（外部服务调用）
    │   └── 结果嵌入[]
    ├── 规则2
    └── ...
```

#### 6.2.2 步骤类型

| 步骤类型 | 说明 | 适用场景 |
|---------|------|---------|
| 内部计算 | 通过条件分支和公式进行计算 | 大部分业务场景 |
| 外部服务 | 调用外部系统获取计算结果 | 个税申报、负工资抵扣等 |

#### 6.2.3 依赖项分类

| 分类 | 说明 | 示例 |
|-----|------|-----|
| 薪酬 | 来自各工资条的薪酬项 | 计件提成、超时加班费 |
| 常乐豆 | 常乐豆相关收入项 | 保底获豆、回头客获豆 |
| 元数据 | 系统统计类数据 | 员工工龄、门店技师数 |
| 配置参数 | 业务配置常量 | 最低工资标准、社保基数上限 |
| 数据对象 | 业务实体及其属性 | 员工.在职状态、门店.地区 |

#### 6.2.4 外部服务

| 服务名称 | 功能 | 返回字段 |
|---------|------|---------|
| TAX（个税申报） | 根据税前收入计算应纳税额 | 税额、税率、应税收入、速算扣除数 |
| 负工资抵扣 | 计算负工资抵扣金额 | 抵扣前余额、本次抵扣金额、剩余余额、状态 |

### 6.3 跨规则依赖机制

为解决规则执行顺序问题，采用「显式声明」机制：

1. **声明式依赖**：每个规则通过 `crossRuleDeps` 数组显式声明依赖的前序规则索引
2. **执行顺序保证**：系统根据声明的依赖关系确定规则执行顺序
3. **循环检测**：保存时检测是否存在循环依赖（如 A→B→C→A），存在则阻止保存
4. **跨规则引用格式**：`规则{n}/{步骤结果名}`，如 `规则1/保底获豆`

### 6.4 数据对象属性

| 数据对象 | 可用属性 | 说明 |
|---------|---------|-----|
| 员工 | 在职状态、部门、岗位、入职日期、离职日期、负工资余额 | 用于条件判断和业务规则 |
| 订单 | 状态、金额、日期 | - |
| 门店 | 名称、地区 | - |
| 客户 | 名称、等级 | - |

### 7.3 字段说明

#### 7.3.1 基本信息

| 字段 | 必填 | 说明 |
|-----|-----|------|
| 策略名称 | 是 | 标识策略用途，如「全职员工保底获豆计算」 |
| 关联发放策略 | 是 | 选择该依赖计算策略对应的薪酬发放策略 |
| 有效期 | 是 | 策略的有效时间范围 |

#### 7.3.2 规则配置

| 字段 | 必填 | 说明 |
|-----|-----|------|
| 规则名称 | 否 | 标识规则用途，支持自定义输入 |
| 计算频率 | 是 | 周期（日/周/月自然月/月业务周）+ 节点（次月M1日等） |
| 依赖项 | 是 | 该规则计算所需的原始数据，支持多分类选择 |
| 跨规则依赖 | 否 | 显式声明依赖的前序规则索引 |
| 取整模式 | 否 | 四舍五入/向上取整/向下取整，默认四舍五入 |
| 数据格式 | 否 | 保留小数位数，默认0位 |

#### 7.3.3 步骤配置

| 字段 | 必填 | 说明 |
|-----|-----|------|
| 薪酬项名称 | 是 | 计算结果的名称，如「保底收入项」 |
| 计算方式 | 是 | 内部计算：使用公式计算；外部服务：调用外部接口 |
| 条件分支 | 否 | 支持多分支 if-else，每个分支可配置独立公式 |
| 公式 | 是 | 计算公式，支持 SUM/AVG/MAX/MIN/ROUND 及四则运算 |

#### 7.3.4 条件配置

| 字段 | 必填 | 说明 |
|-----|-----|------|
| 字段 | 是 | 可选：依赖项、前序步骤、其他规则步骤、数据对象属性 |
| 判断符 | 是 | 等于/不等于/大于/大于等于/小于/小于等于/包含 |
| 值类型 | 是 | 上下文（系统参数）/引用（引用其他字段）/输入（手动输入）/选择（下拉选项） |
| 筛选值 | 是 | 根据值类型不同而不同 |

#### 7.3.5 结果嵌入

| 字段 | 必填 | 说明 |
|-----|-----|------|
| 来源 | 是 | 选择计算步骤作为结果来源 |
| 别名 | 是 | 嵌入到工资条时的显示名称 |
| 类型 | 是 | 薪酬项（增加收入）/扣缴项（减少收入） |
| 支付方式 | 是 | 现金/常乐豆 |
| 工资条 | 是 | 目标工资条 |
| 分组 | 是 | 目标分组（如应发收入、代扣税等） |

### 7.4 外部服务配置

#### 7.4.1 TAX 个税服务

| 字段 | 说明 |
|-----|------|
| 服务 | 选择「申报个税」 |
| 传入值 | 选择引用前序哪个步骤的结果 |
| 返回字段 | 税额/税率/应税收入/速算扣除数 |

#### 7.4.2 负工资抵扣服务

| 字段 | 说明 |
|-----|------|
| 服务 | 选择「负工资抵扣」 |
| 传入值 | 选择引用前序哪个步骤的结果（一般为收入合计） |
| 返回字段 | 抵扣前余额/本次抵扣金额/剩余余额/状态 |

### 7.5 验证规则

| 验证项 | 规则 |
|-------|-----|
| 策略名称 | 不能为空 |
| 关联发放策略 | 不能为空 |
| 有效期 | 开始日期不能大于结束日期 |
| 步骤结果名称 | 每个规则至少有一个步骤填写结果名称 |
| 循环依赖 | 保存时检测循环依赖，存在则阻止并提示 |
| 跨规则引用 | 仅能引用 `crossRuleDeps` 中声明的前序规则 |

## 8. 页面交互

### 8.1 规则概览区域

- 显示所有规则的可折叠概览
- 点击概览可展开/收起规则详情
- 支持上移/下移调整规则顺序
- 概览文本格式：`依赖项名称 → 第一个步骤结果名 → 第一个嵌入项名称`

### 8.2 条件分支

- 每个步骤支持最多 5 个条件分支
- 分支内条件之间为「且」关系
- 无条件分支作为默认分支（else）
- 公式中可直接引用其他规则的步骤结果，格式：`规则{n}/{结果名}`

### 8.3 公式快捷操作

- 显示函数快捷按钮：求和、平均值、最大值、最小值、取整
- 显示四则运算符号：+、-、*、/
- 显示括号快捷按钮：()、,
- 显示跨规则引用标签，点击可插入引用文本

## 9. 数据结构

### 9.1 策略 Model

```typescript
interface Strategy {
  id: number
  name: string                          // 策略名称
  strategyId: number                    // 关联发放策略ID
  strategyName: string                  // 关联发放策略名称
  startDate: string                     // 开始日期 YYYY-MM-DD
  endDate: string                       // 结束日期 YYYY-MM-DD
  rules: Rule[]                         // 计算规则列表
}

interface Rule {
  name: string                          // 规则名称
  expanded: boolean                     // 是否展开
  frequencyCycle: string                // 计算周期
  frequencyNode: string                  // 计算节点
  dependItems: DependItem[]             // 依赖项
  crossRuleDeps: number[]               // 跨规则依赖（规则索引）
  roundingMode: string                   // 取整模式
  decimalPlaces: string                 // 小数位数
  steps: Step[]                         // 步骤列表
  embeds: Embed[]                       // 嵌入项列表
}

interface DependItem {
  id: number | string                   // ID
  name: string                          // 名称
  payslip?: string                      // 工资条（薪酬类）
  paymentType?: string                  // 支付方式
  value?: string                        // 配置值（配置参数类）
  type?: string                        // 元数据类型
  object?: string                       // 数据对象标识
}

interface Step {
  stepType: 'internal' | 'external'    // 计算方式
  resultAlias: string                   // 结果名称
  expanded: boolean                     // 是否展开
  branches: Branch[]                   // 条件分支
  // 外部服务相关
  externalService?: string             // 服务类型
  externalInput?: number               // 传入值引用步骤索引
  externalResultField?: string          // 返回字段
}

interface Branch {
  conditions: Condition[]              // 条件列表
  formula: string                       // 计算公式
}

interface Condition {
  field: string                         // 字段
  operator: string                      // 判断符
  valueType: string                     // 值类型
  value?: string                        // 手动输入值
  refField?: string                     // 引用字段
  contextParam?: string                 // 上下文参数
  selectValue?: string                  // 下拉选项值
}

interface Embed {
  sourceStep: number                    // 来源步骤索引
  name: string                          // 别名
  resultType: 'salary' | 'deduction'    // 类型
  paymentType: 'cash' | 'bean'          // 支付方式
  targetPayslip: number | string         // 目标工资条
  targetGroup: number | string           // 目标分组
}
```
