# 业务权限需求场景反馈（业务管理者视角）

## 1. 组织权限基础 - 部门、岗位与角色

- **物理性权限因素**：用户进入组织后，基本权限因素已确定
  - 所属部门
  - 所属岗位

- **岗位与角色关系**：
  - 一个人通常只有1个岗位
  - 但可获得多个岗位角色（根据实际工作要求）
  - 岗位角色关联对应的权限

## 2. RBAC与ABAC权限结合

### 2.1 RBAC权限获取
- 人员通过岗位角色获得RBAC权限
- 提供基础的功能操作权限

### 2.2 ABAC权限属性
- 大部分属性可根据人员进入组织后获得
- 示例属性：
  - 所属组织
  - 管理范围
  - 业务环节

### 2.3 组织内数据访问规则
- **默认规则**：本人只能查看自己的数据
- **部门负责人**：可查看部门内所有人员的数据

## 3. 跨组织业务权限场景

### 3.1 非直接关联的业务权限
- 示例场景：授予人员查看门店订单的权限
- 特点：门店订单与内部组织无直接关系
- 原因：从公司业务管理角度，需要该组织参与业务流程中的特定环节

### 3.2 单车业务示例
- **业务背景**：某组织负责单车调配
- **单车状态**：骑行中、正常锁定、维修中、待调配等
- **权限规则**：该组织人员只能查看"待调配"状态的单车相关数据

## 4. 组织内分工与权限范围

### 4.1 层级权限范围
- **组织负责人**：可查看该组织授予管理的最大范围数据（如西南区域）
- **平级人员**：分别管理不同子区域
  - 运营专员A：负责西南区域内的四川省成都市
  - 运营专员B：负责西南区域的重庆市

### 4.2 权限验证流程
1. **ABAC级数据可见性验证**：确认用户可查看哪些数据
2. **RBAC级功能操作验证**：确认用户可执行哪些操作（如调配）
3. **前提条件**：用户需具备当前模块的RBAC页面和功能权限

## 5. 业务流程与权限设计

### 5.1 流程驱动的权限设计
- **设计视野**：从岗位职责切入，设计业务流程
- **流程中的权限**：在流程中加入岗位权限角色
- **角色权限**：角色本身需同时获得RBAC和ABAC权限
- **人员参与**：当具体人员获得岗位角色时，自然参与到相关流程中

## 6. 高效权限设置模式思考

### 6.1 当前权限配置模式
- 针对"调配"功能权限的接口数据
- 为岗位角色单独配置ABAC属性要求（如关联单车订单的"待调配"状态）

### 6.2 高效设置模式探索
- **问题**：如何更高效地设置权限？
- **思考方向**：
  - 业务流程与权限的自动关联
  - 基于业务环节的权限模板
  - 动态权限计算模型
  - 权限继承与组合机制
  - 业务属性与权限规则的自动映射

## 7. 总结与需求要点

### 7.1 核心需求
1. 结合RBAC与ABAC权限模型
2. 支持组织内层级数据访问
3. 实现跨组织业务权限管理
4. 基于业务流程设计权限
5. 提供高效的权限配置方式

### 7.2 设计挑战
1. 复杂业务场景下的权限规则管理
2. 权限属性的动态获取与计算
3. 流程驱动的权限自动化配置
4. 权限模型的可扩展性与灵活性

### 7.3 期望目标
1. 满足复杂业务场景的权限需求
2. 提高权限管理效率
3. 降低权限配置错误率
4. 支持业务快速变化与扩展
