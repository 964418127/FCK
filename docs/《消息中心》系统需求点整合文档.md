# 《消息中心》系统需求点整合文档

### 一、系统核心定位



* 负责消息模板制定、发放计划管理，为低代码平台提供组件接入能力，支持流程中自动调用消息组件发送消息

* 承担按计划发放消息、历史消息存储与查询、定时消息管理等核心任务

### 二、消息通知管理模块

#### 2.1 历史消息管理



* **核心字段**：


  * 基础信息：模板编号、用户 UID、消息主题、消息内容、消息通道

  * 状态信息：状态（成功 / 失败 / 已撤销（仅预生成定时消息））

  * 时间信息：计划发送时间、实际发送时间

  * 来源信息：消息计划（关联消息计划 ID、计划主题）/ 业务触发（关联业务场景 Code、场景描述）

* **操作功能**：


  * 管理操作：查看失败日志、失败重发、删除

  * 查询功能：提供多条件查询入口

#### 2.2 定时消息管理（业务场景触发明确定时发送）



* **核心字段**：


  * 基础信息：模板编号、用户 UID、消息主题、消息内容、消息通道

  * 时间信息：计划发送时间

  * 来源信息：业务触发（关联业务场景 Code、场景描述）

* **操作功能**：


  * 管理操作：撤销发送

  * 查询功能：提供多条件查询入口

### 三、消息发送计划（批量）模块

#### 3.1 核心定义



* 制定消息 “何时发送、如何发送、给谁发送” 的策略，发送时实时生成消息内容

* 支持即时发送、定时发送两种模式

#### 3.2 核心字段



* 计划信息：计划编号、计划主题（明确计划主要目的）

* 目标人群：人群圈选（按身份、角色、标签筛选）

* 关联配置：模板编号、消息主题（核心内容，如 “预约失效”）、消息内容

* 发送配置：发送时间（即时 / 定时）

* 计划状态：计划中 / 待发送（计划发送前 10 分钟推送入消息队列）/ 发送中（到达时间后队列执行）/ 已发送

#### 3.3 操作功能



* 基础操作：新增计划、编辑、删除

* 发送操作：模拟发送（输入指定 UID 测试）、触发发送、查看发送明细

* 管理操作：撤销发送

### 四、配置中心模块

#### 4.1 模板配置（定义消息内容格式）



* **核心字段**：


  * 基础标识：模板编号、模板标题

  * 类型配置：模板类型（短信 / 站内信 / App 推送）、消息类型（验证码 / 通知 / 营销）

  * 内容配置：消息主题、消息内容（支持富文本编辑器（适配邮件）、变量（如`{userName}`、`{orderId}`，业务系统调用时传入变量值））、签名（消息落款）

  * 扩展配置：Deep Link（可选，按通道差异化支持）


    * 短信：消息主题 12121 + 链接

    * 站内信：“查看并处理”+ 业务详情页链接

    * App 推送：消息主题 12121 + 示例链接（如`myapp://hr/salary?action_id=xyz123`）

  * 状态配置：模板状态（启用 / 禁用），禁用后业务系统无法调用

* **操作功能**：新增模板、编辑、模拟发送、启 / 禁用切换

#### 4.2 消息通道配置（支持的发送渠道）



* 已支持通道：站内信、App 推送、邮件、微信模板消息、短信

* 通道关联：模板配置时可绑定对应支持的通道

#### 4.3 规则配置（限制 “何时发、如何发、给谁发”）



* **核心字段**：


  * 规则标识：规则编号、规则名称、规则描述

  * 规则配置：


    * 频次限制：同用户 X 时间内最多接收 N 条相同类型消息；同模板 Y 时间内最多发送 M 条

    * 间隔限制：静默期（间隔期），同用户接收相同类型消息至少间隔 X 时间

    * 范围限制：黑白名单

    * 路由规则：按消息类型配置通道优先级


      * 营销类：站内信 > App 推送 > 短信

      * 验证码类：短信 > App 推送 > 邮件

      * 通知类：站内信 > App 推送 > 短信

  * 关联配置：关联模板（如模板 01、模板 02）

* **操作功能**：新增规则配置、启 / 禁用、查看详情、删除

#### 4.4 消息类型配置（统一管理）



* 支持类型：验证码、通知、营销、其他

* 作用：为模板配置、规则配置提供统一的类型维度筛选与管理

### 五、低代码组件接入支持



* 组件能力：提供消息发送组件，支持低代码平台接入

* 触发方式：支持业务流程中节点触发式发送，自动调用消息组件完成发送任务
