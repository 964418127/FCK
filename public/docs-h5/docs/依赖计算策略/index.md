## 什么是依赖计算策略？

### 核心概念

依赖计算策略用于定义 **薪酬项之间的计算依赖关系**。例如：当员工的月度收入低于最低工资标准时，公司需要补足差额。这个「补足」的计算就依赖「收入」这个前置结果。

| 项目 | 内容 |
|-----|------|
| 🎯 解决什么问题 | 解决薪酬计算中多工资条联动、跨周期计算、外部服务依赖等复杂场景 |
| 👤 谁来配置 | 薪酬管理员，无需开发介入 |
| 📋 配置什么内容 | 计算规则、依赖项、条件分支、外部服务、结果嵌入目标 |

## 产品架构

```
发放策略 → 依赖计算策略 → [规则1, 规则2, 规则3] → 嵌入工资条
```

- **发放策略**：关联的薪酬策略
- **依赖计算策略**：核心配置单元
- **规则**：依赖项 + 公式、跨规则依赖、外部服务
- **嵌入工资条**：计算结果 → 目标工资条的指定分组

---

## 核心概念

| 概念 | 说明 | 示例 |
|-----|------|-----|
| 依赖项 | 计算所需的原始数据 | 计件提成、超时加班费、员工.在职状态 |
| 计算步骤 | 由条件分支和公式组成的计算单元 | 月度收入 = 求和(计件提成, 加班费) |
| 跨规则依赖 | 当前规则引用前序规则的结果 | 规则4引用规则1-3的预扣税 |
| 外部服务 | 调用外部系统获取计算结果 | TAX 申报、负工资抵扣 |
| 结果嵌入 | 将计算结果写入目标工资条 | 保底获豆 → 常乐豆工资条 → 应发收入 |

## 依赖项分类

### 薪酬

来自各工资条的薪酬项，属于现金收入类。

计件提成、超时加班费、节假日加班费、超产值补贴、客户打赏、基础提成、绩效补贴、夜班补贴、回头客激励、周时长激励

### 常乐豆

常乐豆相关收入项，以「常乐豆」为计量单位。

保底获豆、回头客获豆、内部介绍获豆、培训获豆、顶班获豆、住宿获豆、服务圈获豆、营销折扣获豆、三级合伙人获豆

### 元数据

系统统计类数据，由系统自动计算生成。

员工工龄、门店技师数、本月出勤天数、当前余额、累计收益、门店评分、客户满意度、入职日期、业务周期开始、业务周期结束

### 配置参数

业务配置常量，由管理员在系统中维护。

最低工资标准（默认2360）、社保基数上限（默认30000）、公积金基数上限（默认25000）、个税起征点（默认5000）、加班费率（默认1.5）、节假日费率（默认2.0）、保底月工时（默认176）、常乐豆兑换比率（默认1:1）

### 数据对象

| 对象 | 可用属性 |
|-----|---------|
| 员工 | 在职状态、部门、岗位、入职日期、离职日期、负工资余额 |
| 订单 | 状态、金额、日期 |
| 门店 | 名称、地区 |
| 客户 | 名称、等级 |

### 新增/编辑策略

**基本信息配置**：

| 字段 | 说明 |
|-----|------|
| 策略名称 | 必填，输入策略的中文名称，如「全职员工保底获豆计算」 |
| 关联发放策略 | 必填，下拉选择关联的薪酬发放策略，决定策略的适用范围 |
| 有效期 | 必填，选择策略的生效开始日期和结束日期 |

**规则概览区域**：

| 功能 | 说明 |
|-----|------|
| 规则名称 | 点击可编辑，输入规则的简短描述名称 |
| 规则摘要 | 自动显示「依赖项 → 步骤结果 → 嵌入目标」的简要说明 |
| 上移/下移 | 调整规则的执行顺序，影响跨规则依赖的执行先后 |
| 删除规则 | 删除整条规则（需保留至少一条规则） |

**计算频率**：

| 字段 | 选项说明 |
|-----|---------|
| 计算周期 | 日（自然日）、周（自然周）、月（自然月）、月（业务周） |
| 计算节点 | 日结算：次日；周结算：次周周一~周日；月结算：当月 M1日、次月 M1日 |

**依赖项配置**：

| 分组 | 可用项 |
|-----|-------|
| 薪酬 | 计件工资条、补贴工资条、兼职工资条的所有薪酬项 |
| 常乐豆 | 保底获豆、回头客获豆等所有常乐豆收入项 |
| 元数据 | 员工工龄、门店技师数、本月出勤天数等 |
| 配置参数 | 最低工资标准、社保基数上限、个税起征点等 |
| 数据对象 | 员工（在职状态、部门等）、订单、门店、客户 |

**跨规则依赖**：当当前规则需要引用前序规则（如规则1-3）的计算结果时使用。

**计算公式（步骤）**：

| 配置项 | 说明 |
|-------|------|
| 结果名称 | 定义计算结果的变量名，如「月度收入」「保底获豆」 |
| 计算方式 | 内部计算（使用公式）、外部服务（调用外部系统接口） |
| 条件分支 | 支持多分支判断，包含字段、判断符、值类型、条件关系 |
| 公式 | 支持函数（SUM、AVG、MAX、MIN、ROUND）和运算符（+、-、*、/） |

**结果嵌入**：

| 字段 | 说明 |
|-----|------|
| 来源步骤 | 选择哪个计算步骤的结果需要嵌入 |
| 别名 | 在目标工资条中显示的名称 |
| 类型 | 薪酬项（作为收入计入）、扣缴项（作为扣除项目） |
| 支付方式 | 现金、常乐豆 |
| 目标工资条 | 选择嵌入到哪个工资条 |
| 目标分组 | 选择嵌入到工资条的哪个分组（应发收入、代扣税等） |
