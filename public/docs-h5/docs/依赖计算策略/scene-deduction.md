# 场景三：负工资抵扣

## 业务场景

### 什么是负工资

负工资是指员工历史工资结算中出现多发或少扣的情况，形成对公司的欠款。当员工**在职**时，从**现金工资条**抵扣；当员工**离职**时，从**常乐豆工资条**抵扣。

| 项目 | 内容 |
|-----|------|
| 在职员工 | 从现金工资条收入中扣除（每次最多20%） |
| 离职员工 | 从常乐豆工资条收入中扣除 |
| 外部服务 | 负工资服务提供余额查询和抵扣计算 |

## 外部服务：负工资抵扣

### 服务说明

负工资服务是外部系统提供的接口，用于查询员工负工资余额并计算每次抵扣金额。

| 传入参数 | 返回字段 | 说明 |
|---------|---------|-----|
| 收入金额 | before | 抵扣前负工资余额 |
| （当前工资条收入） | deduction | 本次抵扣金额 |
|  | remaining | 剩余未抵扣余额 |
|  | status | 状态（正常/已清零/部分抵扣） |

// 规则3：加班工资条抵扣（离职员工用常乐豆抵扣）

负工资抵扣 = 外部服务(负工资服务, 常乐豆收入)

## 配置要点

| 规则 | 依赖项 | 数据对象条件 | 结果嵌入 |
|-----|-------|-------------|---------|
| 规则1：计件工资条抵扣 | 计件提成、加班费、员工 | 员工.在职状态 = 在职 | 计件工资条 → 代扣税 |
| 规则2：补贴工资条抵扣 | 超产值补贴、客户打赏、员工 | 员工.在职状态 = 在职 | 补贴工资条 → 代扣税 |
| 规则3：常乐豆抵扣 | 保底获豆、员工 | 员工.在职状态 = 离职 | 常乐豆工资条 → 代扣税 |

### 跨规则依赖说明

规则2依赖规则1，规则3依赖规则1和规则2，通过 `crossRuleDeps` 显式声明。系统会按依赖顺序执行，确保负工资余额在规则间传递。

## 场景小结

**负工资抵扣**展示了数据对象和外部服务的结合能力：

- 通过 **数据对象条件**（员工.在职状态）判断走不同分支
