场景一 全职员工保底获豆

🏦 保底获豆计算

当全职员工月度收入低于最低工资标准时，由公司补足差额，以常乐豆形式发放

业务场景

💡 业务背景

为保障全职员工的基本生活，当员工某月的现金收入（计件提成 + 加班费）低于当地最低工资标准时，差额部分由公司以「常乐豆」形式补足。

适用对象
全职员工（按月发放薪酬）
发放形式
常乐豆（非现金）
计算周期
月（自然月）

业务流程

计算逻辑

// 公式定义
月度收入 = 计件提成 + 超时加班费 + 节假日加班费

if (月度收入 < 最低工资标准):
    保底获豆 = 最低工资标准 - 月度收入
else:
    保底获豆 = 0

// 示例计算
// 情况1: 计件=3000, 加班=500, 节假日=300
// 月度收入 = 3000 + 500 + 300 = 3800
// 3800 >= 2360 → 保底获豆 = 0

// 情况2: 计件=1500, 加班=200, 节假日=0
// 月度收入 = 1500 + 200 + 0 = 1700
// 1700 < 2360 → 保底获豆 = 2360 - 1700 = 660

✅ 场景小结

保底获豆是最基础的依赖计算场景，体现了依赖计算的核心能力：

  • 多个依赖项（计件 + 加班费）通过 求和公式 汇总
  • 通过 条件分支 判断是否需要补足
  • 结果嵌入到 常乐豆工资条 的应发收入分组
