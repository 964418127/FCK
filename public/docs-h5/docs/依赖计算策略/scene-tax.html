场景二 兼职员工个税计算

💰 兼职员工个税计算

兼职员工按业务周预扣个税，月末汇总进行退补，涉及多条规则和跨规则引用

业务场景

💡 业务背景

兼职员工的薪酬按业务周（周一开始）计算，每周预扣 3% 个税。月末时，将当月各周的预扣税与实际应纳税额进行对比，多退少补。

适用对象
兼职员工（按周发放薪酬）
预扣税率
3%
计算周期
月（业务周）

业务流程

跨规则依赖机制

⚠️ 关键设计

规则4需要引用规则1-3的计算结果（各周的预扣税），这里采用显式声明的跨规则依赖机制，而不是简单的「按顺序引用」。

声明方式
规则4配置 crossRuleDeps: [0, 1, 2]，显式声明依赖规则1-3
引用格式
公式中使用 规则1/第1周预扣个税 格式引用
执行顺序
系统根据声明确定先执行R1-R3，再执行R4

计算逻辑

// 规则1-3：每周预扣税
第1周预扣 = (基础提成 + 绩效补贴 + 夜班补贴) × 3%
第2周预扣 = (回头客激励 + 服务时长补贴) × 3%
第3周预扣 = (周时长激励 + 节假日补贴) × 3%

// 规则4：月末汇总退补
税前收入合计 = 基础提成 + 回头客激励 + 周时长激励
实际税额 = 外部服务(TAX, 税前收入合计)
预扣税合计 = 规则1/第1周预扣个税 + 规则2/第2周预扣个税 + 规则3/第3周预扣个税

if (预扣税合计 > 实际税额):
    个税退税 = 预扣税合计 - 实际税额  → 嵌入第4周工资条(应发收入)
else:
    个税补缴 = 实际税额 - 预扣税合计  → 嵌入第4周工资条(代扣税)

配置要点

规则 依赖项 跨规则依赖 结果嵌入
规则1
第1周预扣		 基础提成、绩效补贴、夜班补贴 无 第1周工资条 → 代扣税
规则2
第2周预扣		 回头客激励、服务时长补贴 无 第2周工资条 → 代扣税
规则3
第3周预扣		 周时长激励、节假日补贴 无 第3周工资条 → 代扣税
规则4
月度汇总		 基础提成、回头客激励、周时长激励 规则1, 2, 3 第4周工资条 → 应发收入/代扣税
✅ 场景小结

个税计算展示了跨规则依赖的核心能力：

  • 通过 显式声明 crossRuleDeps 引用前序规则结果
  • 在公式中使用 规则{n}/{结果名} 格式插入跨规则引用
  • 支持 外部服务调用（TAX申报）获取真实税额
  • 多条规则协同完成 多退少补 的完整业务闭环
