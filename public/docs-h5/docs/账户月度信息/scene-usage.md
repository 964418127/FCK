## 业务描述

查看员工账户的常乐豆使用明细，包括提现记录、消费记录和退货记录。

财务人员或管理员通过账户月度信息的"提现明细"或"消费明细"按钮，跳转到此页面查看对应的使用记录。提现和消费显示负数(-)，退货显示正数(+)。

---

## 业务流程

```
┌─────────────────────────────────────┐
│  从账户月度信息跳转                  │
│  (month, employeeId, employeeName,  │
│   type=withdrawal/consumption)      │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│         筛选条件区域                 │
│  (月份、使用类型、单号、时间...)     │
└──────────────┬──────────────────────┘
               │
               ▼
        [查询] 按钮
               │
               ▼
┌─────────────────────────────────────┐
│         数据表格区域                 │
│  (展示常乐豆使用明细数据)            │
└──────────────┬──────────────────────┘
               │
               ▼
         [导出] 按钮
```

---

## 筛选条件

| 字段 | 说明 | 控件类型 | 业务说明 |
|------|------|---------|---------|
| 月份 | 选择查询月份 | monthPicker | 选择查询的月份，格式YYYY-MM，默认从跳转参数带回 |
| 使用类型 | 选择使用类型 | select | 可选：全部、提现、消费、退货，默认从跳转参数带回（withdrawal/consumption） |
| 单号 | 输入单号 | input | 按单号精确查询某笔使用记录 |
| 发起时间 | 选择开始时间 | datetimePicker | 筛选发起时间范围起点 |
| 完成时间 | 选择结束时间 | datetimePicker | 筛选完成时间范围终点 |
| 姓名&工号 | 支持按姓名或工号搜索 | input | 输入员工姓名或工号进行搜索，支持模糊匹配，默认从跳转参数带回员工姓名 |
| 部门 | 输入部门名称 | input | 按部门维度筛选数据 |
| 岗位 | 输入岗位名称 | input | 按岗位维度筛选数据 |
| 主服务门店 | 主服务门店名称或编码 | input | 按主服务门店筛选 |
| 劳动关系门店 | 劳动关系门店名称或编码 | input | 按劳动关系门店筛选 |
| 合同主体 | 输入合同主体名称 | input | 按合同主体公司筛选 |
| 状态 | 选择状态 | select | 可选：全部、结算发起、结算成功、结算失败、提现冻结、提现成功、提现失败、消费冻结、消费成功、消费失败、退货中、退货成功、退货失败，用于筛选不同状态记录 |
| 账户类型 | 固定选择"余额账户" | select | 当前固定为"余额账户" |

## 表格列

| 列名 | 说明 | 业务说明 |
|------|------|---------|
| 月份 | 数据所属月份 | 格式为 YYYY-MM，用于标识数据统计周期 |
| 使用类型 | 类型标签 | Tag显示：提现(警告色)、消费(主色)、退货(成功色)、结算(信息色) |
| 单号 | 交易单编号 | 唯一标识每笔使用记录的编号 |
| 姓名 | 员工姓名 | 员工基本信息 |
| 部门 | 员工部门 | 员工所属组织部门 |
| 岗位 | 员工岗位 | 员工职位信息 |
| 工号 | 员工工号 | 员工唯一标识 |
| 主服务门店 | 主服务门店名称 | 员工主要服务的门店 |
| 劳动关系门店 | 劳动关系门店 | 员工签订劳动合同的门店或公司 |
| 合同主体 | 合同签约主体公司 | 员工签署劳动合同的公司主体 |
| 数量 | 交易数量 | 提现和消费显示负号(-)，退货显示正号(+)，单位为常乐豆 |
| 交易前余额 | 交易前账户余额 | 执行该笔交易前的账户余额 |
| 交易后余额 | 交易后账户余额 | 执行该笔交易后的账户余额，高亮显示 |
| 账户类型 | 账户类型 | 统一显示为"余额账户" |
| 发起时间 | 交易发起时间 | 发起交易的时间戳 |
| 完成时间 | 交易完成时间 | 交易完成的时间戳 |
| 交易金额 | 实际交易金额 | 实际发生的交易金额 |
| 状态 | 当前状态 | Tag显示，根据使用类型不同显示不同状态：提现类/消费类/退货类各有不同状态 |

## 操作按钮

| 按钮 | 功能 | 详细说明 |
|------|------|---------|
| 查询 | 根据条件搜索 | 点击后根据筛选条件重新查询并展示常乐豆使用明细数据 |
| 重置 | 清空筛选条件 | 清空所有筛选条件输入框，恢复默认状态，表格数据重置为默认展示 |
| 导出 | 导出数据 | 导出搜索条件下所有符合条件的数据，导出字段包括：月份、使用类型、单号、姓名、部门、岗位、工号、主服务门店、劳动关系门店、合同主体、数量、交易前余额、交易后余额、账户类型、发起时间、完成时间、交易金额、状态，导出格式为 Excel |

---

## 跳转参数

| 参数 | 来源 | 说明 |
|-----|------|------|
| month | URL Query | 月份，格式 YYYY-MM，从账户月度信息页面带入 |
| employeeId | URL Query | 员工工号，从账户月度信息页面带入 |
| employeeName | URL Query | 员工姓名，从账户月度信息页面带入 |
| type | URL Query | 使用类型：withdrawal(提现)、consumption(消费)，从账户月度信息页面带入 |

## 分页说明

每页显示条数可选：10、20、50、100 条，支持跳转到指定页码。
