场景二 单条路线更新

✏️ 单条路线更新

针对特定路线手动更新间隔时间和标准时间，适用于员工反馈路线变化后的单独修正

业务场景

💡 业务背景

当员工反馈某个路线的实际出行情况与系统记录不符时（如新开了一条隧道、某条道路限行），调度人员需要对特定路线进行单独更新。这个场景主要用于处理偶发的、个别的路线数据修正。

触发方式
员工反馈、巡店发现、地图提示
操作方式
选择单条路线，更新耗时数据
适用范围
个别路线的临时修正

操作流程

配置要点

配置项 配置值 说明
更新方式 单条更新 一次处理一条路线
导航策略 可自定义 可重新选择导航方式和方案策略
数据来源 地图API 实时调用地图服务获取最新数据
标准时间 可手动调整 基于推荐规则，可手动修正
更新说明 必填 记录本次更新的原因

更新策略说明

⚠️ 更新策略选择

在更新时需要选择该路线的后续更新策略：

更新策略 说明
手动更新 该路线不参与自动更新，需要人工手动触发更新。适用于路线变化频繁、需要人工确认的场景。
自动更新 该路线将按照全局配置的自动更新规则定期调用地图API同步数据。适用于数据稳定、不需要频繁调整的路线。
✅ 场景小结

单条路线更新是针对个别路线数据修正的便捷操作，主要特点：

  • 操作简单，一键更新
  • 支持重新配置导航策略
  • 可灵活设置手动/自动更新
  • 必须填写更新说明，便于追溯
