三 需求背景

💡 业务痛点

推拿师需要在多个门店之间移动服务客户，但门店之间的距离和出行时间难以准确预估。当技师从一个门店转移到另一个门店时，如果不知道准确的出行时间，可能会导致客户等待时间过长或排班冲突。

🎯 解决什么问题
建立门店路线间隔时间数据库，精确管理推拿师在任意两个门店之间的出行耗时
👤 谁来使用
排班调度人员、薪酬管理员
📋 配置什么内容
门店路线数据、导航策略、标准间隔时间、自动更新规则

四 核心概念

概念 说明 示例
门店路线 从一个起始门店到终点门店的单向路线 天府凯德店 → 金牛凯德店
导航方式 出行采用的交通方式 步行、骑行、骑电动车、公交、驾车
方案策略 地图导航的路线选择策略 距离最短、用时最短
标准间隔时间 用于排班计算的预留时间，基于导航耗时上浮一定比例 导航耗时18分钟 → 标准间隔20分钟（+11%）
自动更新 定时调用地图API更新路线数据 每日凌晨2:00自动更新

六 功能说明

门店路线间隔时间库包含三个核心功能页面：列表页、新增路线、编辑路线。

列表页面功能

功能区域 功能说明
搜索筛选 支持按城市、起始门店、终点门店进行筛选查询
数据表格 展示路线、起始门店、终点门店、导航策略、距离、导航耗时、标准间隔时间、更新方式、最后更新时间等字段
批量更新耗时 点击「批量更新耗时」按钮，一次性更新多条路线的间隔时间
新增路线 点击「新增门店路线」按钮，新增成对双向路线
更新单条 点击操作栏的「更新」按钮，进入单条路线更新对话框
查看历史 点击操作栏的「历史」按钮，查看该路线的更新历史记录
分页 支持10/20/50/100条每页的分页切换

列表字段说明

字段 说明
城市 门店所属城市，支持成都市、北京市、上海市、广州市、深圳市
路线 格式为「起始门店 → 终点门店」，显示路线方向标签（单向）
起始门店 显示门店名称、地址、经纬度坐标
终点门店 显示门店名称、地址、经纬度坐标
导航策略 显示当前使用的导航方式（步行/骑行/骑电动车/公交/驾车）和方案策略（距离最短/用时最短）
距离 两点之间的直线或路线距离，单位为公里(km)，保留2位小数
导航耗时 基于地图API实时计算的实际出行时间，单位为分钟
标准间隔时间 用于排班计算的预留时间，单位为分钟，通常为导航耗时的110%-150%
更新方式 自动 系统定时任务自动更新
手动 人工手动更新
最后更新时间 该路线数据最近一次更新的时间

新增路线功能

⚠️ 重要特性

新增路线时会自动创建一对双向路线：前往路线和返回路线。例如新增「天府凯德店 → 金牛凯德店」会自动同时创建「金牛凯德店 → 天府凯德店」。

新增字段说明

字段 说明
城市 必填，选择门店所属城市，用于筛选可选门店列表
起始门店 必填，选择路线起点门店，支持搜索过滤
终点门店 必填，选择路线终点门店，支持搜索过滤
前往路线导航方式 步行 骑行 骑电动车 公交 驾车
前往路线方案策略 距离最短 按照实际行驶距离选择路线
用时最短 按照预计时间选择路线
返回路线导航方式 与前往路线相同，可独立配置
返回路线方案策略 与前往路线相同，可独立配置
路线详情 点击「开始计算」后显示，基于地图API实时计算：
  • 距离：路线总距离（公里）
  • 导航耗时：预估出行时间（分钟）
  • 标准时间：可手动调整的建议间隔时间
  • 更新策略：手动更新/自动更新
  • 路线说明：可选的备注信息

标准时间推荐规则

导航耗时区间 推荐标准间隔时间
0-10分钟 导航耗时 × 1.5（+50%）
11-20分钟 导航耗时 × 1.2（+20%）
21-30分钟 导航耗时 × 1.15（+15%）
31-60分钟 导航耗时 × 1.1（+10%）
60分钟以上 导航耗时 + 10分钟

配置定时任务自动调用地图API更新所有已标记为「自动更新」的路线数据。

字段 选项说明
启用自动更新 开启 启用定时更新任务
关闭 禁用自动更新

开启自动更新后，系统会显示自动更新预览，清晰告知下次执行的时间和规则。

七 数据结构

门店路线数据

interface StoreInterval {
  id: string;                           // 路线唯一标识
  route: string;                        // 路线名称，如"天府凯德店 → 金牛凯德店"
  direction: 'forward' | 'backward';    // 路线方向
  startStore: Store;                   // 起始门店
  endStore: Store;                      // 终点门店
  navigationType: NavigationType;      // 导航方式
  planStrategy: PlanStrategy;          // 方案策略
  distance: number;                    // 距离(km)
  navigationTime: number;              // 导航耗时(分钟)
  standardTime: number;                // 标准间隔时间(分钟)
  updateStrategy: 'manual' | 'auto';   // 更新策略
  lastUpdateType: 'auto' | 'manual';   // 上次更新方式
  lastUpdateTime: string;              // 上次更新时间
  history: UpdateHistory[];           // 更新历史
}

interface Store {
  id: string;                          // 门店ID
  name: string;                         // 门店名称
  displayName: string;                  // 显示名称
  address: string;                       // 门店地址
  city: string;                         // 所属城市
  longitude: string;                    // 经度
  latitude: string;                     // 纬度
}

interface UpdateHistory {
  updateTime: string;                   // 更新时间
  oldNavigationTime: number | null;    // 原导航耗时
  newNavigationTime: number;            // 新导航耗时
  oldStandardTime: number | null;       // 原标准时间
  newStandardTime: number;             // 新标准时间
  updateNote: string;                   // 更新说明
  updaterType: 'system' | 'user';      // 更新者类型
  updaterName: string;                 // 更新者名称
}

type NavigationType = 'walking' | 'cycling' | 'e-bike' | 'bus' | 'driving';
type PlanStrategy = 'shortest-distance' | 'shortest-time';

自动更新配置

interface AutoUpdateConfig {
  enabled: boolean;                     // 是否启用
  frequency: 'daily' | 'weekly' | 'monthly';  // 更新频率
  time: string;                        // 更新时间，HH:mm格式
  dayOfWeek?: number;                  // 星期几（周频），1-7
  dayOfMonth?: number;                 // 日期（月频），1-31
}
