## 需求背景

### 业务痛点

推拿师需要在多个门店之间移动服务客户，但门店之间的距离和出行时间难以准确预估。当技师从一个门店转移到另一个门店时，如果不知道准确的出行时间，可能会导致客户等待时间过长或排班冲突。

| 项目 | 内容 |
|-----|------|
| 🎯 解决什么问题 | 建立门店路线间隔时间数据库，精确管理推拿师在任意两个门店之间的出行耗时 |
| 👤 谁来使用 | 排班调度人员、薪酬管理员 |
| 📋 配置什么内容 | 门店路线数据、导航策略、标准间隔时间、自动更新规则 |

## 核心概念

| 概念 | 说明 | 示例 |
|-----|------|-----|
| 门店路线 | 从一个起始门店到终点门店的单向路线 | 天府凯德店 → 金牛凯德店 |
| 导航方式 | 出行采用的交通方式 | 步行、骑行、骑电动车、公交、驾车 |
| 方案策略 | 地图导航的路线选择策略 | 距离最短、用时最短 |
| 标准间隔时间 | 用于排班计算的预留时间，基于导航耗时上浮一定比例 | 导航耗时18分钟 → 标准间隔20分钟（+11%） |
| 自动更新 | 定时调用地图API更新路线数据 | 每日凌晨2:00自动更新 |

### 新增路线

**重要特性**：新增路线时会自动创建一对双向路线：前往路线和返回路线。

| 字段 | 说明 |
|-----|------|
| 城市 | 必填，选择门店所属城市 |
| 起始门店 | 必填，选择路线起点门店 |
| 终点门店 | 必填，选择路线终点门店 |
| 导航方式 | 步行、骑行、骑电动车、公交、驾车 |
| 方案策略 | 距离最短、用时最短 |
| 标准时间 | 基于导航耗时自动上浮计算 |

**标准时间推荐规则**：

| 导航耗时区间 | 推荐标准间隔时间 |
|-------------|-----------------|
| 0-10分钟 | 导航耗时 × 1.5（+50%） |
| 11-20分钟 | 导航耗时 × 1.2（+20%） |
| 21-30分钟 | 导航耗时 × 1.15（+15%） |
| 31-60分钟 | 导航耗时 × 1.1（+10%） |
| 60分钟以上 | 导航耗时 + 10分钟 |

## 数据结构

```typescript
interface StoreInterval {
  id: string;                           // 路线唯一标识
  route: string;                        // 路线名称
  direction: 'forward' | 'backward';    // 路线方向
  startStore: Store;                     // 起始门店

  navigationTime: number;               // 导航耗时(分钟)
  standardTime: number;                 // 标准间隔时间(分钟)
  updateStrategy: 'manual' | 'auto';     // 更新策略
  lastUpdateTime: string;               // 上次更新时间
  history: UpdateHistory[];              // 更新历史
}
```
